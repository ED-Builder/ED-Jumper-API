# ED-Jumper API

后端 API 服务，基于 Cloudflare Workers + D1 构建，为 ED-Jumper 提供链接分发管理、用户认证与权限控制能力。

## 功能概览

- 管理员与普通用户双角色体系
- 用户注册、登录、个人资料管理
- 链接分发创建与编辑，支持密码保护
- 分发验证状态：管理员验证 / 自行验证 / 未验证
- 管理员可管理用户权限（登录、创建、验证、改资料）
- 权限变更实时通知弹窗

## 快速开始

```bash
npm install
npm run dev
```

## 部署到 Cloudflare Workers

### 1. 创建 D1 数据库

在 Cloudflare 控制台创建一个 D1 SQLite 数据库，然后执行 `schema.sql` 中的初始化语句。

### 2. 配置 wrangler.toml

编辑 `wrangler.toml`，将 `database_id` 和 `database_name` 替换为你的 D1 数据库信息。

### 3. 设置环境变量

| 变量 | 说明 |
|------|------|
| `AUTH_SECRET` | 会话签名密钥，使用足够长且随机的字符串 |
| `ADMIN_PASSWORD_HASH` | 管理员登录密码的 SHA256 值 |
| `ALLOWED_ORIGINS` | 允许访问 API 的前端源站，多个地址用英文逗号分隔 |

### 4. 部署

连接到 Cloudflare Workers，构建命令填 `npm ci`，在绑定中添加 D1 数据库，变量名必须为 `DB`。

## 权限系统

注册时默认获得全部四项权限：

| 权限 | 说明 |
|------|------|
| 登录鉴权 | 允许登录系统 |
| 创建分发 | 允许创建和编辑链接分发 |
| 验证分发 | 允许自行验证分发 |
| 更改个人信息 | 允许修改用户名、头像和密码 |

权限共存规则：
- 取消登录鉴权 → 其余三项自动取消
- 取消创建分发 → 验证分发自动取消

管理员可在后台修改用户权限，变更后用户将在前端收到通知弹窗。

## 分发验证状态

| 状态 | 图标 | 条件 |
|------|------|------|
| 管理员验证 | `/moderator.svg` | 管理员创建，或被管理员验证的分发 |
| 自行验证 | `/verified.svg` | 用户自行标记为已验证的分发 |
| 未验证 | `/unknown.svg` | 用户未验证的分发 |

管理员可以在控制台中修改每个分发的验证状态。

## 技术说明

- 用户密码与分发密码均使用 SHA256 哈希存储
- 会话令牌基于 HMAC-SHA256 签名，有效期 15 分钟
- 权限变更通过 `permission_notifications` 表记录，前端轮询展示弹窗
